# Docs Quality Checklist

Use this checklist before opening or approving docs/distribution PRs.

## Content Accuracy
- [ ] API names and parameters match current public package.
- [ ] Snippets reflect current behavior and fallback semantics.
- [ ] No unsupported or unverifiable claims.

## Evidence
- [ ] Each claim includes a code/docs/test path.
- [ ] External article statements are marked verified or needs-proof.
- [ ] Comparison language remains factual and neutral.

## Readability
- [ ] Headings are clear and scenario-driven.
- [ ] Snippets are minimal and directly useful.
- [ ] Terminology is consistent across README and docs.

## Governance
- [ ] Change is mapped to a task ID (R-01 ... R-18).
- [ ] Reviewer assignment follows contributor lanes.
- [ ] Any deferred topic remains out of scope.
